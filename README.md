# Workflow

The application is built using nextjs, and it is deployed on Github Pages. The application is built using TypeScript, and it uses Tailwind CSS for styling.

Use Github Actions to build and deploy the application to Github Pages.
Build the application, then deploy it to the `gh-pages` branch of the repository.

The website should be responsive and should work well on both desktop and mobile devices.

Use a simplistic design for the website, with a focus on showcasing the company's services and portfolio. The website should be easy to navigate and should provide clear information about the company and its services.

Use a color scheme that matches kalalib_logo.png.

## Nav Bar:
It should be collapsible on mobile devices, and should include the following links:
1. Left:
    - KALA (Use kalalib_logo.png as the logo) -> ./

2. Right:
    - KALA Solutions -> https://advisory.kalalib.com
    - KALA Studio -> https://studio.kalalib.com
    - Team -> /team
    - Contact -> /contact

# Pages

## Landing Page:
- URL: `/` (index.html)
- Description: The landing page of the website, which provides an overview of the company and its services.

The Landing page should include the following sections:
1. Hero section: A section that introduces the company and its services, with a call-to-action button that directs users to the services page.
2. Services section: A section that highlights the different services offered by the company, with links to the services page for more information.
3. Portfolio section: A section that showcases some of the projects that the company has worked on, with links to the portfolio page for more information.
4. Contact section: A section that provides contact information for the company, including an email address and a contact form for users to get in touch with the company.

## Team:
- URL: `/team` (team.html)
- Description: A page that provides information about the team members and their roles within the company.


Refer to `agent/` for more details on the content of the pages.
